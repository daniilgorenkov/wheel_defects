# Wheel Defects

Проект для бинарной классификации дефектов колесной пары по сигналу вертикального ускорения (`accz`) и скорости (`spd`) от датчика УМДВ.

## Что делает проект

- Читает сырые CSV-файлы из папки `data/*`.
- Фильтрует и сегментирует сигнал по оборотам колеса.
- Формирует выборки переменной длины.
- Обучает нейросетевую модель (по умолчанию `DeepThreeHeadModel`).
- Логирует метрики, кривые и артефакты в ClearML.
- Сохраняет лучший чекпоинт в `checkpoints/best_model.pt`.

## Структура проекта

```text
wheel_defects/
├── config.py                  # Конфиги обучения, препроцессинга и модели
├── run.py                     # Точка входа для подготовки данных и обучения
├── requierments.txt           # Зависимости (имя файла с опечаткой)
├── data/
│   ├── v1/, v2/               # Сырые CSV-файлы
│   ├── sample.csv             # Пример формата входных данных
│   └── prep_data.pkl          # Сериализованные выборки (создается автоматически)
├── checkpoints/
│   └── best_model.pt          # Лучший чекпоинт (создается при обучении)
└── src/
    ├── mixins/                # Препроцессинг, датасеты, тренер, логирование
    └── models/                # Набор моделей
```

## Требования

- Python 3.10+
- Linux/WSL (рекомендуется, проект сейчас ориентирован на такую среду)
- Доступ к ClearML серверу (если хотите логировать эксперименты)

## Установка

1. Создайте и активируйте виртуальное окружение:

```bash
python -m venv .venv
source .venv/bin/activate
```

2. Установите зависимости:

```bash
pip install -r requierments.txt
```

## Формат данных

Ожидается CSV с разделителем `;` и колонками:

- `datatime` — timestamp измерения
- `accz` — ускорение по оси Z
- `spd` — скорость

Минимальный пример:

```csv
datatime;accz;spd
2024-05-18 09:33:52.680;-0.7;2.33
2024-05-18 09:33:52.700;0.6;2.33
```

### Как формируется таргет

Сейчас таргет берется из имени файла:

- класс `1`, если предпоследний токен имени файла содержит `17`
- иначе класс `0`

Пример: `20240608_062318_0017_acc.csv -> 1`.

## Препроцессинг

В `PreprocessorMixin` выполняются этапы:

1. Чтение данных и перевод `datatime` в datetime.
2. Фильтрация по скорости (по умолчанию от 11 до 30).
3. Нормализация сигнала.
4. Разбиение на обороты колеса по накопленной дистанции.
5. Дополнительная фильтрация по амплитуде ускорения.
6. Сохранение подготовленных выборок в `data/prep_data.pkl`.

Ключевые параметры находятся в `PreprocessorConfig` внутри `config.py`.

## Запуск

Точка входа: `run.py`.

Важно: в текущей реализации флаг `--train` означает обучение на уже сохраненных подготовленных данных (`data/prep_data.pkl`).

### 1) Полный цикл (препроцессинг + обучение)

```bash
python run.py
```

### 2) Обучение только на готовых выборках

```bash
python run.py --train
```

## Обучение и логирование

- Обучение запускается через класс `Trainer`.
- Используется `FocalLoss(gamma=2.0)`.
- В `TrainerConfig` настраиваются:
  - `lr`, `weight_decay`, `epochs`, `patience`
  - параметры ранней остановки по переобучению
  - `monitor_metric` и `monitor_mode`
- В ClearML логируются:
  - train/val метрики
  - confusion matrix
  - PR/ROC-кривые
  - распределения вероятностей
  - артефакт с путем к лучшей модели

## Где менять модель

По умолчанию в `run.py` используется:

- `model=DeepThreeHeadModel`

Доступные альтернативы в `src/models/`:

- `Baseline`
- `LiteBaseline`
- `CNNTransformerEncoder`
- `ThreeHeadModel`
- `DeepThreeHeadModel`

Чтобы переключить архитектуру, измените блок создания `ModelBuilder` в `run.py`.

## Полезные замечания

- Файл зависимостей называется `requirements.txt`.
- В `PreprocessorConfig.USE_COLS` используется имя `datatime` — входные CSV должны содержать именно такую колонку.
- Лучший чекпоинт сохраняется в `checkpoints/best_model.pt`.

## Быстрый чек перед запуском

- В `data/` лежат подпапки с CSV-файлами.
- В CSV есть колонки `datatime`, `accz`, `spd`.
- Установлены зависимости из `requirements.txt`.
- Активировано виртуальное окружение.
- Если нужен ClearML логгинг, настроены переменные/конфиг ClearML.
